# План: Права доступа на уровне аналитик

## Цель
Пользователь может получить доступ не ко всему листу, а к конкретному подразделению (записи аналитики).
Например: "Иванов видит только Потребительский кредит на листе BaaS.1".

## Шаги

### 1. БД: таблица analytic_record_permissions
- [x] `CREATE TABLE analytic_record_permissions (id, user_id, analytic_id, record_id, can_view, can_edit)`
- [x] record_id = конкретная запись аналитики (подразделение)
- [ ] Если запись = parent группы → доступ ко всем children

### 2. Бэкенд: CRUD для analytic_record_permissions
- [x] GET /api/users/{user_id}/analytic-permissions — список прав
- [x] PUT /api/users/analytic-permissions — обновить права

### 3. UI прав: аналитики в панели пользователей
- [x] Секция "Доступ к аналитикам" с деревом Модель > Аналитика > Записи
- [x] Чекбоксы Просмотр/Ввод на каждую запись

### 4. Фронт: автофиксация подразделения
- [x] GET /users/{id}/allowed-records/{sheet_id} — возвращает разрешённые записи
- [x] При открытии листа: если 1 запись → автоматический pin
- [x] Пользователь сразу видит "свои" показатели
